# 常见问题

## Workflow

Read the answer, take the next action, and avoid over-planning.

---

## Do I need Claude Code, Codex, or Cursor?

No. FirstMVPSkill adapts to the agent you already have.

Start with one primary agent. Use advanced playbooks only when the task requires them.

---

## Do I need to read every file?

No.

Start with:

```text
config/default-mode.md
skills/first-mvp-launch/SKILL.md
```

---

## Why 1-3 MVP features?

Because beginners usually fail by adding too much too early.

Three features are enough to show value and small enough to finish.

---

## Why not use multi-agent immediately?

Multi-agent work adds coordination cost.

Use it only after:

- Scope is frozen
- Files are assigned clearly
- You can review outputs

---

## What if my agent does not support slash commands?

Use the plain prompt fallback from [错误处理](troubleshooting.md).

---

## Is this a code framework?

No. It is a Skill Pack and execution workflow.

It helps you decide what to build, command agents, review results, fix blockers, and launch.

---

## Do Not

- Do not wait until you understand every file.
- Do not add advanced playbooks on Day 1.
- Do not treat the 7-day plan as a large roadmap.
