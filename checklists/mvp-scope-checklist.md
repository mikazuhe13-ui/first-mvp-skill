# MVP Scope Checklist

> Check before you build. Keep your MVP tiny.

---


## Workflow

Use the sections in this file in order. Keep the output compact, preserve MVP scope, include constraints, and end with one next action.

---
## The 1-3 feature rule

Your MVP has 1 to 3 features. Never more than 3.

- [ ] I have 1-3 features (never more than 3)
- [ ] Each feature can be described in one sentence
- [ ] Each feature has a clear user action and output

---

## Feature check

For each feature, answer:

### Feature 1: [name]
- [ ] Can I build this in 1 day?
- [ ] Is this essential for the product to work?
- [ ] Can I describe it in one sentence?

### Feature 2: [name] (optional)
- [ ] Can I build this in 1 day?
- [ ] Is this essential for the product to work?
- [ ] Can I describe it in one sentence?

### Feature 3: [name] (optional)
- [ ] Can I build this in 1 day?
- [ ] Is this essential for the product to work?
- [ ] Can I describe it in one sentence?

---

## Tool check

- [ ] I have 2-4 tools maximum
- [ ] Each tool has a clear job
- [ ] All tools are ones I already know
- [ ] No "nice to have" tools

---

## Exclusion check

- [ ] I have a written "Not in V1" list
- [ ] Each excluded feature has a reason
- [ ] I'm not secretly planning to build excluded features

---

## Time check

- [ ] 7 days is enough for all features
- [ ] Each day has a clear deliverable
- [ ] I have buffer time (Day 5) for fixes

---

## Red flags

If you checked any of these, simplify:

- [ ] "I need more than 3 features" → Cut to 3
- [ ] "Each feature needs sub-features" → Simplify features
- [ ] "I need to learn new tools" → Use what you know
- [ ] "This will take more than 7 days" → Cut scope
- [ ] "I can't describe it in one sentence" → Simplify

---

## Scoring

Count your checkmarks:
- 15+ checks: Your MVP is too complex. Simplify.
- 10-14 checks: Good scope. Proceed.
- <10 checks: You haven't thought enough. Fill in the gaps.

## Do Not

- Do not add scope beyond this file's purpose.
- Do not skip acceptance gates, evidence, or the next action.
- Do not switch tools unless the user explicitly asks or the current tool cannot complete the task.
